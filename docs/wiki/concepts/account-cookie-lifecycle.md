# 概念: 账号与 Cookie 生命周期

## 核心目标

在尽量减少验证码调用的前提下，保证每个账号尽可能完成签到。

## 生命周期阶段

1. 装载阶段
   - 从环境变量或 Docker Cookie 文件读取 `NS_COOKIE`。
   - 从 `USER/PASS` 与 `USERn/PASSn` 读取账号密码。
2. 对齐阶段
   - 账号数组与 Cookie 数组按最大长度补齐，形成索引对应关系。
3. 执行阶段
   - 优先使用旧 Cookie 签到。
   - 无效时使用对应账号密码登录获取新 Cookie。
4. 刷新阶段
   - 用新 Cookie 再次签到。
   - 成功后更新内存中的 Cookie 列表。
5. 持久化阶段
   - 全部账号处理完毕后统一写回（文件 / 青龙 / GitHub Variables）。

## 关键收益

- 避免每次都触发登录与验证码。
- 多账号场景下减少互相影响，单账号失败不阻断其他账号。
- 环境无关的统一业务逻辑，环境差异只在持久化接口层处理。
